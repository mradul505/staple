🗄️ Exercise A: Database Design

Schema Design

Technologies: PostgreSQL (SQL) + Elasticsearch (NoSQL)

PostgreSQL Schema:
CREATE TABLE compensations (
  id SERIAL PRIMARY KEY,
  employer VARCHAR(255),
  job_title VARCHAR(255),
  location VARCHAR(255),
  annual_base_pay DECIMAL(12,2),
  total_compensation DECIMAL(12,2),
  years_experience_industry INTEGER,
  years_experience_current_company INTEGER,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
Elasticsearch Mapping:
{
  "properties": {
    "employer": { "type": "keyword" },
    "jobTitle": { "type": "text", "fields": { "keyword": { "type": "keyword" } } },
    "location": { "type": "text", "fields": { "keyword": { "type": "keyword" } } },
    "annualBasePay": { "type": "double" },
    "totalCompensation": { "type": "double" },
    "yearsExperienceIndustry": { "type": "integer" },
    "createdAt": { "type": "date" }
  }
}

🔌 Exercise B: GraphQL API

API Endpoints

Technology: GraphQL with filtering, sorting, and field selection

Example Queries:
📝 List with Filters & Sorting:
query {
  compensations(
    filter: {
      annualBasePay: { min: 120000 },
      locationContains: "San Francisco"
    },
    sort: [{ field: ANNUAL_BASE_PAY, direction: DESC }],
    pagination: { limit: 10, offset: 0 }
  ) {
    data {
      id employer jobTitle location 
      annualBasePayFormatted
    }
    totalCount
  }
}
🎯 Single Record with Sparse Fields:
